QMA Boolean Minimization is a C++ library and command-line tool that simplifies Boolean functions using the Quine–McCluskey algorithm. Given any Boolean function expressed as a truth table output string, the tool produces the minimal sum-of-products (SOP) representation — the simplest possible Boolean equation that matches your truth table exactly.

## What the tool does

You provide a string of `0`s and `1`s representing the output column of a truth table, one character per row. The tool parses that string, derives the number of variables from its length, groups minterms by the number of set bits, and iteratively combines them into prime implicants. It then selects the minimal cover using an Essential Prime Implicants (EPI) table, with a brute-force fallback for cases not fully resolved by the EPI step.

The result is a single Boolean equation printed to standard output.

**Key constraints:**

- The input string length must be a power of 2 (2, 4, 8, 16, 32, 64, …). The tool infers the number of variables as log₂(length).
- Only `0` and `1` characters are supported. Don't-care values (`X`) are not currently handled in the documented interface.

## What you get

The output is a sum-of-products Boolean expression. Variables are named alphabetically starting from `A`, with `!` denoting logical NOT (negation) and `+` denoting logical OR between product terms.

For example, a 5-variable minimization might return:

```text
A!BCE+B!CD+!AB!C+AB!DE+!A!BD
```

Each term is a product (AND) of literals. A bare letter means the variable is uncomplemented; `!` before a letter means it is complemented. The `+` operator combines terms in a logical OR. Variables that do not appear in a term are eliminated — they have no effect on that term's output.
